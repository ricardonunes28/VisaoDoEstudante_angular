Componente

Decorator que marca uma classe como um componente Angular e fornece metadados de configuração que determinam como o componente deve ser processado, instanciado e usado em tempo de execução.

Module


O módulo é como um grande contêiner que contém um ou muitos contêineres pequenos chamados Componente, Serviço.
Os módulos são úteis para agregar funcionalidades em comum dentro de uma aplicação. Por exemplo, é possível criar um módulo para conter todas as funcionalidades da área de administração de sua aplicação usando o comando ng generate module.

Um componente contém:
  • Modelo HTML ou código HTML

  • Código (TypeScript)

  • Serviço: é um código reutilizável compartilhado pelos Componentes, de modo que não é necessário reescrever o código

Diretivas


Diretivas estruturais são responsáveis por geralmente manipular o DOM, adicionando ou removendo elementos. Podemos reconhecê-los através do asterisco (*) que segue com o atributo da diretiva.
Por exemplo, o Angular nos provê algumas diretivas estruturais, como: ngFor, ngIf e ngSwitchCase.


Rotas


As rotas são definidas por dois elementos principais, o Path, que é o caminho (URL) e o Componente, que é o componente a ser carregado. Ainda existem outras formas de carregar páginas mais eficientes, como carregamento assíncrono, via módulos, mas este é assunto para um próximo artigo.

Service


Podemos que dizer que no arquivo service da nossa estrutura de projeto, é aonde contem toda a parte logica que precisamos.
Service é o objeto usado para organizar e/ou compartilhar estados de objetos e as regras de negócio da aplicação. Ele é singleton, ou seja, há apenas uma instância disponível durante a vida útil da aplicação.
